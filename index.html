About this sandbox

A general purpose starter that stays in a single file.
No framework required
All in one file
Works with GitHub Pages

The goal of this project is to show how far a basic GitHub Pages setup can go before any complex tooling appears. There is no build command, bundler or dependency list. The browser handles everything.

This makes the sandbox a safe place for experiments. It can hold notes, UI sketches, interactive elements and ideas that might later be turned into separate projects or real applications.

  • Readable structure. The page is split into sections such as hero, about, playground, gallery and roadmap, so it is easy to navigate the code.
  • Good habits. Theme toggling, keyboard shortcuts and small animations are added to hint at practices used in more advanced projects.
  • Future friendly. Any section can be removed, duplicated or rewritten for another idea without touching build tools.

Sliders and buttons that reshape a grid in real time.

In focus mode a small group of cells stays bright and everything else fades back. This feels similar to highlighting the current task while the rest of the work stays in context.
Wave mode animates brightness across the grid. This imitates a simple loading or activity indicator that could sit behind content.
Random mode treats the grid like a tiny particle field. Cells switch on and off at random which can create subtle motion for a background.

Suggested learning roadmap

A simple path that starts at this file and grows from there.

The sandbox is meant to be a safe place for small steps. It is ok to treat it as a notebook that just happens to live on the web.

Nothing here requires deep knowledge of JavaScript or design. Curiosity and experimentation are enough to start.

Commit small changes
Read the HTML structure
Modify text and colors
Add a new card or section
Step 1 - Understand layout
Short term
Read the hero, section and card markup to see how the layout is built. Try changing one heading or paragraph and commit the update.
Step 2 - Adjust styles
Next stage
Modify colors, spacing or fonts inside the CSS. Small tweaks can create a very different feel while keeping the structure the same.
Step 3 - Explore scripts
When comfortable
Look at the script block at the end of the file. Follow how event listeners connect to sections such as the playground and command palette.
Step 4 - Branch out
Later
Copy this file into a new repository for a completely different idea. Replace content step by step until it fits the new purpose.

Command palette and shortcuts

A simple overlay inspired by modern editor palettes.

The command palette is a small overlay that can trigger common actions. It appears on top of the page, listens for input and reacts to a selected item.

This version can jump to sections or toggle the theme. It also shows how keyboard shortcuts can be wired into the page without extra libraries.

Press Ctrl + K on Windows or ⌘ + K on macOS to open the palette. Use arrow keys and Enter or click with the mouse.
