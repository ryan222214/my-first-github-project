About this lab

A place to learn GitHub and ship experiments with AI.
AI assisted development
Zero previous Git experience
Content first mindset

This project started as a simple "Hello World" site. It now acts as a small hub that can grow into something more serious as Ryan learns GitHub, version control, and basic web concepts.

The rule is simple. Ryan describes the idea, large language models write the code, and GitHub stores the work in one reliable place. No need to become a traditional software engineer in order to ship useful tools.

  • Goal 1. Use this repo as a sandbox for AI generated tools that support TikTok, YouTube, and book projects.
  • Goal 2. Build the habit of shipping small improvements often instead of waiting for big launches.
  • Goal 3. Learn just enough HTML, CSS, and JavaScript to understand what AI is doing and to tweak it.

GitHub learning roadmap

From first commit to a small library of AI built tools.

The plan is to keep things simple. Learn skills only when they are needed. Each step is something Ryan can ask AI to help with.

GitHub becomes the home for all experiments. Over time this can turn into a real portfolio of tools that support content, books, and business systems.

Keep everything in repos
Use Git as history
Ship small and often
AI first approach
Phase 1 — Web basics
Now to 2 weeks
Use AI to modify this page, add sections, and style. Learn how HTML, CSS, and JavaScript connect. Practice committing small improvements.
Phase 2 — Local setup
When ready
Connect GitHub to a local editor like VS Code. Let AI generate files on the laptop, then push them here. This unlocks larger projects.
Phase 3 — Real utilities
Next 3 to 6 months
Build small utilities that automate parts of the content workflow. Things like trackers, dashboards, or prompt helpers.
Phase 4 — Products
Long term
Turn the best internal tools into products. Offer them to other creators or students. GitHub becomes the foundation of a real software layer under the content business.

How this page was built

Ryan asked AI to "make the most advanced single page site possible without creating extra files". Everything here lives in one index.html file. That is all GitHub Pages needs.

The next step is to treat this like a living document. Add new sections, track experiments, and let AI improve the visuals over time. Every update is stored as a commit so nothing is wasted.

This gives Ryan a deeply practical way to learn GitHub by doing real work instead of abstract exercises.
